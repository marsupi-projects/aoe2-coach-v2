## Decisions made

### Evaluator/parser mismatch — fixed
Claude generates reports in a multi-line format (`PLAYER 1 — name` header, then
`Civilisation : ...`, `eAPM : ...`, `Result : ...` on separate lines). The original
`_extract_players` regex in `core/parser.py` only matched the single-line format
(`PLAYER: Name | Civ | eAPM: N | Result: Win`). Updated to try single-line first,
fall back to multi-line. Also handles `Civilisation`/`Civilization` spelling and strips
emoji from results (`✅ Win` → `Win`).

### `limited_data` field added to parse_replay
`_c9_data_limitations` in the evaluator checked `parsed_data.get("limited_data", False)`
but `parse_replay` never returned that key. Added `"limited_data": has_ai` to the return
dict so vs-AI games correctly trigger the data limitations criterion check.

### Dummy ChromaDB data removed
Two seeded test entries (`run-001`, `run-smoke-test`) with Frank/Joris fixture data were
polluting player history results. Deleted directly via the ChromaDB client. Two real runs
(`96ea6838`, `e2747d6f`) remain.

### Comments and logging added to main.py and agent.py
Per user request. `agent.py` now prints progress at each stage: run start, each Claude
turn, each tool call and result, evaluation, and final score. `CLAUDE.md` updated to
require this on all future edits to these two files.

### Human-readable report filenames
Reports now named `{game_date}_{civs}_{map}_{result}_{analysis_date}.txt` instead of
`{run_id}.txt`. Example: `20260613_byzantines-aztecs_wade_win_20260615.txt`.
Implementation: `build_report_filename()` added to `tools.py`; `_last_parsed` module-level
cache in `tools.py` gives `tool_write_coaching_report` access to replay metadata without
changing the tool API; `agent.py` computes `report_path` using the same function.
`parse_replay` now also returns `game_date` (YYYYMMDD from `match.timestamp`).

### Evaluator reading the wrong text — fixed
`agent.py` was evaluating `final_text` (Claude's closing turn summary, e.g. "I've
completed all tasks...") rather than the coaching report itself. Claude spreads work
across multiple turns and the report is written in an earlier turn, not the last.
Fixed by reading the report file directly if it exists; `final_text` is only used as a
fallback when the file is absent.

### Unicode arrows broke Windows console — fixed
`→` (U+2192) in print statements caused `UnicodeEncodeError` on Windows cp1252.
Replaced with plain ASCII `->` throughout `agent.py`.

### README.md created
Covers: requirements, venv setup, `.env` configuration, how to run `main.py`, sample
terminal output (real output from a passing run), output file table, evaluator scoring,
and project structure. `CLAUDE.md` updated to require README updates at end of session.

### venv guidance added
No venv existed. `.venv/` was already in `.gitignore`. README now documents venv creation
and activation. Actual venv not yet created — user will do this before the next session.

## Things explicitly decided against

- **Changing the `write_coaching_report` tool API** to pass filename metadata from Claude:
  used a module-level cache (`_last_parsed`) in `tools.py` instead, keeping the tool
  signature stable and avoiding prompt changes.
- **Increasing the 120s timeout** in SPEC.md §4: the structural fix (pending_tool_results
  flag, session 03) is sufficient; the limit stands.
