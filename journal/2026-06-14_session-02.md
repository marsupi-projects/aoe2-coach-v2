## Decisions made

### `core/agent.py` — manual agentic loop
- Used the manual agentic loop pattern (not the SDK tool runner) because we need fine-grained control: per-tool logging, `parse_replay` error capture for the evaluator, `tool_calls_seen` tracking, and enforced timeout. The tool runner abstracts these away.
- Used `thinking={"type": "adaptive"}` — supported by Sonnet 4.6 and appropriate for multi-step reasoning over replay data.
- `tool_calls_seen: set[str]` tracks whether `write_run_entry` and `store_run_memory` were actually called, feeding the evaluator's `run_meta` dict (criteria c10 and c11).
- `parsed_data` is captured from the `parse_replay` tool result (if it returns a dict) rather than re-parsed from the message history, keeping the evaluator input clean.
- The entire agentic loop and post-loop work is wrapped in a `try/except` that calls `log_run_error` before re-raising, so every fatal failure is captured in the structured JSONL log.
- Tool errors (including `parse_replay`'s `NotImplementedError`) are caught per-tool, serialised as `"ERROR: ..."` strings, and returned to Claude as `tool_result` blocks with `error=True` in the logger. Claude sees the error and is expected to adapt (note data limitations, produce what it can).
- `log_run` (feedback.py) is called with `human_players` only — AI players are filtered by `is_human` before any persistence call.
- `CLAUDE_MODEL` is taken from `config.py` (defaults to `claude-sonnet-4-6`, overridable via `.env`).

### `infra/logger.py` — structured step-level JSONL logging
- Five event types: `run_start`, `tool_called`, `tool_result`, `run_end`, `run_error`.
- Writes to `logs/YYYY-MM-DD.jsonl` using `LOGS_PATH` from `config.py`.
- Tool results are truncated to 500 chars to keep log files manageable.
- `error: bool` flag on `tool_result` entries makes failed tool calls grep-able without parsing content.

### `main.py` — CLI entrypoint
- Accepts a single positional argument: the replay file path.
- Validates the file exists before calling `agent.run()`.
- Prints report path, evaluator score, run ID, and elapsed time on completion.
- Error paths propagate as exceptions (non-zero exit from unhandled exception).

## Things explicitly decided against

- **Streaming in the agentic loop**: streaming adds complexity to the loop (chunked content reassembly) for a single-user local tool. Non-streaming `client.messages.create()` is simpler and correct here. Revisit if timeouts become a problem on slow network connections to the Anthropic API.
- **Calling `log_run` from inside the agentic loop**: deferred until after `evaluate()` so the evaluator score is always present in the run entry.
- **Argparse**: `sys.argv` direct access is sufficient for v1.1 (one positional argument). Argparse would be appropriate if flags or multiple arguments are added later.
