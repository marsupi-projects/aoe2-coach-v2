### conftest.py for environment isolation

`tests/conftest.py` sets `ANTHROPIC_API_KEY=test-key` at module level before any
imports. This is the only way to satisfy `config.py`'s import-time `os.environ["ANTHROPIC_API_KEY"]`
without requiring a real key during tests. The `tmp_chroma` fixture patches
`core.database._client` with a `PersistentClient` pointing at `tmp_path/chroma`,
isolating each test from production data.

### ChromaDB empty metadata rejection

ChromaDB rejects `metadatas=[{}]` — it requires each metadata dict to be non-empty.
Discovered during the first test run; fixed by using `{"source": "test"}` as
the default metadata in test seed helpers.

### c6 raw string collision

The first draft of `test_7_of_11_fails` used `"No historical context here."` as the
raw string. This accidentally contained "historical context" as a substring, so c6
passed instead of failing. Fixed by using a raw string with completely neutral
content that doesn't trigger either c6 keyword.

### pytest added to requirements.txt

`pytest>=7.0` added so `pip install -r requirements.txt` in a fresh venv includes
the test runner.

### README updated

Added "Running the tests" section with the pytest command and a note on test count
and isolation.

## Things explicitly decided against

- **Mocking ChromaDB**: Used real PersistentClient with a tmp_path instead.
  Avoids the risk of mock/real divergence that has burned this project before
  (session 04 evaluator-reads-wrong-text bug was a mismatch of the same kind).
- **test_tools.py and test_agent.py**: Not written this session. Both require
  either mocked Anthropic API calls or real API access; left for a later session
  once the core modules are stable.
