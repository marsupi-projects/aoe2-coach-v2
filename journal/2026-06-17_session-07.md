## Decisions made

### reflect.py validated on real data

`python reflect.py` ran successfully against 7 scored runs. The reflection report
(`data/reflections/reflection_2026-06-16.md`) produced 6 concrete, grounded conclusions:

- `historical_context_present` and `report_content` are the most persistent failure criteria
- walter2515 consistently outperforms dreas.kesbeke in eAPM (24–26 vs 16–17) — future
  reports should address this gap explicitly every time this pairing appears
- The same two player pairings account for all 6 multi-player runs — agent can anticipate
  this context
- Scores jumped from 0.545–0.727 to 1.000 for identical player pairings, confirming the
  failure criteria are fully solvable

The reflection also picked up "Frank" from old dummy JSONL entries — not harmful but noted.

v1.3 is fully validated end-to-end.

### Reflection reports moved from logs/ to data/reflections/

`logs/` is for step-level JSONL debug traces (`logger.py`). Reflection reports are
meaningful agent output for the user to read — they belong under `data/` alongside
coaching reports.

Changes:
- Added `REFLECTIONS_PATH` to `config.py` (default: `data/reflections`)
- Added `REFLECTIONS_PATH` to `.env.example`
- Updated `learning/reflection.py` to use `REFLECTIONS_PATH` instead of `LOGS_PATH`
- Updated tests to monkeypatch `REFLECTIONS_PATH`
- Updated `README.md`, `BLUEPRINT.md` references
