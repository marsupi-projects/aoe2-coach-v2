### v1.3 — reflection.py

`learning/reflection.py` implements:
- `load_run_history()` — reads all JSONL from `data/runs/`, deduplicates by
  `run_id` (prefers the entry with `evaluator_score != null`)
- `_format_entries()` — formats run entries as readable lines for the Claude prompt;
  handles both old `eAPM` and new `eapm` field names
- `reflect(min_runs=3)` — calls Claude (non-streaming, no tools) to identify patterns,
  stores conclusions in ChromaDB collection `"reflections"`, writes markdown report to `logs/`
- `_write_report()` — writes `logs/reflection_YYYY-MM-DD.md`

Triggered manually: `python reflect.py`

### v1.3 — prompt_tuner.py

`learning/prompt_tuner.py` implements:
- `score_by_version()` — groups scored runs by `prompt_version`, computes avg
  score and pass rate per version
- `best_version()` — returns the version with the highest avg score
- `tune()` — prints the full analysis and the promotion instruction

Promotion is manual (print instruction, no auto-edit) to prevent accidental
regressions from overwriting `core/prompt.py` without review.

### get_all() added to core/database.py

Non-semantic retrieval: returns all documents in a collection up to a limit.
Used by `get_reflections()` because reflections are sorted by recency, not by
semantic similarity to a query. Returns flat lists (not nested), unlike `query()`.

### get_reflections() added to memory/retrieval.py

Reads all documents from the `"reflections"` ChromaDB collection via `get_all()`,
sorts by `metadata.date` descending, returns the most recent n.

### core/prompt.py — reflections injected into task prompt

`build_task_prompt` now accepts an optional `reflections: list[dict]` parameter.
When reflections are present, they are included under a `RECENT REFLECTIONS` header
before the main task instruction, giving Claude accumulated patterns as context.

### core/agent.py — reflections retrieved before first API call

`get_reflections(n=3)` is called at the start of each run. If reflections exist,
they are passed to `build_task_prompt` and printed. If none exist, a hint is
printed telling the user to run `reflect.py`.

### reflect.py root entrypoint

`python reflect.py` runs reflection then prompt tuner in sequence. Exits cleanly
if there are not enough scored runs yet (< 3).

## Things explicitly decided against

- **Auto-promoting the best prompt version**: `prompt_tuner.py` prints the promotion
  instruction but does not edit `core/prompt.py` automatically. Prevents silent
  regressions when only one version is on record or the sample size is too small.
- **Semantic search for reflections**: reflections are sorted by recency, not by
  similarity to the current game. A recent reflection about player eAPM trends is
  always relevant; semantic search would retrieve by topic and might miss recent
  conclusions.
- **Running reflection after every main.py run**: reflection is triggered manually.
  Auto-trigger (every N runs) is deferred to v2.1 when the scheduler is active.
